# bhelper

VibeHelpers — приложение на \$mol с ботом, подбором помощников и просмотром вакансий HH.ru.

## Быстрый старт (локально)

```bash
npm start bog/bhelper
# открой http://localhost:9080/bog/bhelper/app/-/test.html
```

## Docker

```bash
docker compose up --build
# приложение будет на http://localhost:9080/bog/bhelper/app/-/test.html
```

Dockerfile собирает project в `/mam/bog/bhelper` и запускает `npm exec mam /bog/bhelper`.

## Структура

- `app/` — главное приложение (\$mol).
- `app/prof/` — список помощников с ролями HH.ru (динамическая фильтрация, переход в бота).
- `app/vaka/` — поиск вакансий HH.ru.
- `app/crm/`, `app/settings/`, `app/lk/` — CRM, настройки, личная карточка.
- `assets/` — иконки/манифест.
- `Dockerfile`, `docker-compose.yml` — контейнеризация.

## Приложение

- Бот открывается со страниц помощников/вакансий, контекст подтягивается из выбранной роли.
- Тему можно переключать (светлая/тёмная), работает оффлайн-установка (\$mol offline).

## Почему \$mol (коротко)

- Быстрый старт: dev-сервер в одной команде, бандлы собираются автоматически.
- Реактивность без коллбеков: mem/act-свойства сами отслеживают зависимости, нет подписок.
- CRUS-DB в браузере: офлайн-чтение/запись, E2E-шифрование, синхронизация дельтами без конфликтов.
- Типизированные view.tree + CSS-in-TS: генерация строго типизированных компонентов и стилей, линковка локалей из коробки.
- Zero-периметр: нет отдельного сервера/ORM/API — данные живут у клиента, права и шифрование заданы в модели.
- Мгновенный деплой: статический билд можно выкладывать хоть на GitHub Pages; TG/VK мини-приложение подключается за полчаса.
- Дизайн под все платформы: адаптивная верстка, темы и компоненты выглядят нативно на вебе/мобиле/десктопе.
- Гибкость: view.tree быстро перекомпоновывает UI без ломки логики; стили статически проверяются.
- Удобство для телефонов: лёгкие бандлы, ленивый рендер, автофокус, жесты — комфортно на мобильных.
- Offline-first: сервис-воркер + CRUS-DB дают полноценную работу без сети, синк дельтами при появлении связи.
- Производительность: реактивный рендер без виртуального DOM, маленькие пакеты — ощущение как у нативной апки.
